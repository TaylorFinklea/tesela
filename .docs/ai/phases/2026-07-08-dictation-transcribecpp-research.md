## The landscape (verified)

- **transcribe.cpp** = the runtime ("transcription.cpp" is a misremembering). github.com/handy-computer/transcribe.cpp — CJ Pais (Handy author), first project of Mozilla.ai Builders-in-Residence. MIT. "llama.cpp for STT": GGUF models on ggml, 16 families / 60+ variants (Parakeet ×10, Canary incl. 180m-flash, Whisper ×12, Nemotron streaming, Moonshine, Voxtral…). Metal auto on Apple Silicon; Vulkan/CUDA/tinyBLAS elsewhere. Single C header; official bindings: Python, TS (dlopen, not WASM), **Rust (`transcribe-cpp` crate, v0.1.2, features metal/vulkan/cuda)**, **Swift/ObjC (Package.swift targets iOS 16+, Metal on device)**. Streaming first-class: `stream_begin/feed/finalize` → **committed + tentative** text with change flags. All handy-computer HF GGUFs are numerically validated + WER-tested vs NeMo reference (Modal GPUs). Age risk: repo Apr 2026, releases ~3 weeks old, Handy v0.9.0 warns "there will be bugs".
- **Handy v0.9.0** (2026-07-01) integration pattern: `transcribe-cpp` crate (metal on macOS), whisper-rs deleted (Whisper GGUFs run through transcribe.cpp), transcribe-rs kept only for legacy ONNX. 65-model catalog.json (capability flags, quants, recommended_rank; **rank 1 = Parakeet Unified EN 0.6B, Q8_0 ~731 MB, streaming**); downloads via hf-hub into the shared HF cache. StreamRouter: recorder pushes 16 kHz frames to a streaming worker; engine "lease" excludes concurrent batch; UI repaints only on committed/tentative change. Silero VAD v4 (15-frame prefill, 15/55-frame hangover). Idle model-unload timer.
- **canary-180m-flash**: 182M FastConformer+Transformer; en/de/fr/es ASR + bidirectional translation; PnC built in; timestamps experimental (NOT exposed by transcribe.cpp v1 port or sherpa-onnx output). **CC-BY-4.0, explicitly commercial-OK** (attribution only). Batch-only, ≤40 s chunks (VAD-segment long form). GGUF 133 MB (Q4_K_M)–208 MB (Q8_0) WER-verified ≈1.9% LS-clean; sherpa-onnx int8 ≈154 MB. Open ASR mean WER 7.12 — beats whisper-large-v3-turbo (7.83, 809M) at 182M; smaller than whisper-small. No newer small Canary exists (canary-1b-v2 = 25-lang 1B sibling). No 180m CoreML port (FluidInference ships canary-**1b-v2**-coreml; FluidAudio Canary engine PR #709 open 2026-06-17).
- **parakeet-unified-en-0.6b** (Apr 2026): ONE checkpoint, offline + buffered streaming (left context recomputed per chunk; not cache-aware). Latency knob 160 ms–2.08 s in 80 ms steps; WER 5.91 offline → 6.52 @0.56 s → 8.44 @0.16 s. English-only, PnC built in. **License: NVIDIA Open Model License (NOT CC-BY-4.0 — changed from tdt-v2/v3)** → review before shipping as default. Streaming runtimes today: transcribe.cpp (offline+streaming from same GGUF — README table stale, docs/models/parakeet-unified-en-0.6b.md is authoritative), sherpa-onnx ≥v1.13.2 (PR #3575 merged 2026-05-09), FluidAudio ≥v0.15.3 (`StreamingUnifiedAsrManager`, CoreML tiers 320 ms/640 ms/1.12 s/2.08 s — 320 ms floor), parakeet-rs (ort; unified ONNX at HF bobNight/parakeet-unified-en-0.6b-onnx). Multilingual streaming alternative (40 locales, cache-aware, CC-checked NVIDIA lineup): **nemotron-3.5-asr-streaming-0.6b** — supported by transcribe.cpp AND FluidAudio v0.15.0+.
- Browser: keep ASR server-side (existing pattern). parakeet.js (WebGPU+WASM) exists; transformers.js has Parakeet CTC only (TDT open #1310). Not worth ~0.5 GB per-browser downloads + COOP/COEP.

## Recommended architecture

- **Server/desktop (the real swap)**: adopt **`transcribe-cpp` Rust crate** as a new engine behind an extracted engine trait (mirror iOS's protocol design), dispatched on catalog `family`. Models from handy-computer HF GGUFs: canary-180m-flash Q8_0 (208 MB, 4-lang, batch default) + parakeet-unified-en-0.6b Q8_0 (731 MB, streaming+batch). Metal ≈ today's whisper-rs. Keep whisper-rs one release as fallback; retire à la Handy once proven (also route Whisper GGUFs through transcribe.cpp). Replace dead .nemo placeholders. Fix en-route: per-entry filename ext (:161), checksum/size verification, per-session model handling replacing the global single-slot mutex.
- **Streaming spine (new)**: WS endpoint (or extended /ws) carrying committed/tentative frames (transcribe.cpp semantics, Handy's change-flag repaint). Web capture is greenfield: getUserMedia → AudioWorklet → 16 kHz mono f32 → WS; wire the dead mic button. Desktop gets it for free (same web bundle).
- **iOS**: bump FluidAudio pin → v0.15.5+ (note: v0.15.5 renamed DownloadUtils→ModelHub) → catalog entry for parakeet-unified + `StreamingUnifiedAsrManager` (CoreML/ANE, 320 ms+); add streaming shape to TranscriptionEngine protocol (AsyncStream of partials + finalize); render partials in CaptureBar/GrCaptureSheet middle slot; resurrect voice.streaming toggle. Canary-180m on iOS: no CoreML port → options: (a) server fallback via Tailscale (works today), (b) transcribe.cpp Swift binding (iOS 16+, Metal — unproven, worth a spike), (c) wait for FluidAudio PR #709 (canary-1b-v2, bigger). Recommend (a) now + (b) spike later. SwiftWhisper retires once GGUF/CoreML paths cover dictation.
- **Web/browser**: stays a server client. No in-browser inference.
